豆瓣 Frodo API 可用性检测

生成带正确 HMAC-SHA1 签名的 Frodo 请求，看当前网络环境下豆瓣服务器是否放行。
用法：换到要测的网络（手机 4G / 家庭宽带 / VPN / 公司 WiFi 等），打开这个页面点"开始检测"即可。

3. 结果

结果怎么看

  • 返回 JSON 含 subject_collection_items 字段，且状态 200 → 通了 ✅，当前网络可以正常抓取
  • {"code": 996, "localized_message": "签名错误"} → 当前 IP 被豆瓣风控（最常见的"浏览器能开、程序 400"就是这种），换网络或等一段时间
  • {"code": 112, "msg": "invalid apikey"} → apiKey 被豆瓣回收了，项目需要换 key
  • {"code": 115, ...} 或提到限流、rate limit → 当前 IP 被限速
  • 浏览器超时、连接失败 → 网络层问题（DNS、代理、防火墙）
  • 页面提示 Failed to fetch 但点"在新标签页打开 URL"能看到 JSON → 是浏览器 CORS 拦截了 fetch，接口实际是通的，按新标签页里看到的 JSON 判断

为什么"浏览器能开链接，程序却 400"？

两种是完全不同的接口：

  • 浏览器打开 m.douban.com/subject_collection/... 是给人看的 HTML 网页，任何人无需鉴权都能访问
  • 监控脚本请求的是 frodo.douban.com/api/v2/...，是豆瓣手机 App 的内部接口，要 apiKey + HMAC 签名 + 特定 UA

豆瓣的反作弊会针对"某 IP + Frodo apiKey 组合"单独限流，返回 "签名错误"作烟雾弹。这个页面用和脚本一模一样的签名算法来复现那个 API 请求，所以它的结果才能代表监控脚本在当前网络下的真实情况。
