# OpenClaw 豆瓣高分监控

## 目标
这个 skill 用来监控豆瓣里“新出现并达到目标门槛”的影视内容，默认关注以下条件：

- 类型属于电影、电视剧、综艺、纪录片、动画，或你配置的其他类型
- 评分大于 `8.0`
- 评分人数大于 `3000`
- 条目是首次达标，或者在冷却后出现了明显增长，值得再次提醒

## 模式
本 skill 当前按**完整模式**设计。

完整模式要求：

- 豆瓣周榜页或聚合页
  使用浏览器抓取
- 豆瓣详情页
  优先使用 HTML 解析
- 宿主环境
  具备浏览器自动化依赖

## 抓取策略
采用双层抓取模式：

1. 候选发现层
   使用移动端浏览器方式打开豆瓣周榜或聚合页，默认覆盖：
   - 电影
   - 华语剧集
   - 全球剧集
   - 国内综艺
   - 国外综艺

2. 详情核验层
   对具体豆瓣详情页优先使用普通 HTML 解析，提取标题、评分、评分人数、年份和链接。

原因：

- 豆瓣移动端周榜页更可能依赖前端渲染，适合浏览器抓取
- 豆瓣详情页通常可以直接从 HTML 里提取评分和评分人数，效率更高

## 数据源策略
采用“豆瓣侧 + TMDB 侧”的混合候选池。

默认候选源：

- 豆瓣移动端多榜单页，默认包括电影、华语剧集、全球剧集、国内综艺、国外综艺
- TMDB 的电影和剧集热门接口，例如 `trending`、`popular`

来源职责：

- 豆瓣周榜页
  用来发现近期在豆瓣社区热起来的内容，使用浏览器方式抓取
- TMDB 热门接口
  用来补充全球范围内刚冒头的电影和剧集
- 豆瓣详情页
  作为评分和评分人数的真值来源，优先使用 HTML 解析

## Cookie 策略

本 skill 允许用户提供豆瓣 cookie，作为可选增强项。

cookie 的作用：

- 提高浏览器抓取周榜页的稳定性
- 降低部分页面被安全校验打断的概率

但需要明确：

- cookie 不能替代限频策略
- cookie 不能保证完全绕过风控
- 如果访问频率过高，登录态账号仍可能被限制

处理原则：

- cookie 只作为可选能力，不应成为唯一前提
- cookie 必须按敏感信息处理
- 默认应通过环境变量或 secret 注入，而不是写入仓库

## 执行流程

1. 读取上一次状态文件；如果不存在，就用空状态开始。
2. 使用浏览器打开豆瓣多榜单页，提取候选条目链接。
3. 补充 TMDB 热门接口中的候选条目。
4. 统一字段并尽可能补全 `douban_id` 和豆瓣详情页链接。
5. 按 `douban_id` 或回退键去重。
6. 访问豆瓣详情页，优先用 HTML 提取标题、评分、评分人数和年份。
7. 把候选分成“当前已达标”和“继续观察”。
8. 与历史状态比对，判定新增命中和二次关注。
9. 更新状态文件和监控库。
10. 输出报告。

## 核心规则

### 首次达标
条目满足以下条件时，视为“新增命中”：

- 当前评分大于 `min_rating`
- 当前评分人数大于 `min_rating_count`
- 并且之前没有状态记录，或之前从未达标

### 冷却机制
避免同一条目反复提醒。

- 默认至少要求距离上次提醒超过 `realert_cooldown_days`

### 值得二次关注
只有在条目已经命中过、且冷却窗口已过时，才判断是否值得再次提醒。满足以下任一条件即可：

- 当前评分比上次提醒时至少高出 `rating_delta_for_realert`
- 当前评分人数比上次提醒时至少多出 `rating_count_delta_for_realert`
- 当前评分人数跨过了 `milestone_counts` 中配置的里程碑

## 入库规则
不要把所有发现到的条目都永久加入监控库。只有具备一定潜力的条目，才值得后续继续检查。

入库前至少要满足这些身份条件：

- 有标题
- 有来源链接或稳定外部 id
- 类型已知，或能够归一化到内部类型集

并且至少满足以下任一条件：

- 出现在豆瓣榜单页
- 出现在选定的 TMDB 热门源
- 豆瓣评分大于等于 `7.5`
- 豆瓣评分人数大于等于 `1000`
- 之前已经在监控库里，且仍然处于观察窗口内

## 退库规则
出现以下任一情况时，把条目归档：

- 超过观察窗口仍然从未达标
- 豆瓣评分长期低于 `7.5`
- 最近 `7 天` 的评分人数增长低于 `100`
- 连续 `14 天` 没再出现在豆瓣榜单和 TMDB 热门源里
- 豆瓣详情页连续多次访问失败

## 来源优先级
当多个来源字段不一致时，优先级如下：

1. 豆瓣详情页
2. 豆瓣周榜页
3. TMDB 元数据

TMDB 负责扩充候选和补展示字段，但不应该覆盖豆瓣评分和豆瓣评分人数。
