# OpenClaw 豆瓣高分监控

## 目标
这个 skill 用来监控豆瓣里“新出现并达到目标门槛”的影视内容，默认关注以下条件：

- 类型属于电影、电视剧、综艺、纪录片、动画，或你配置的其他类型
- 评分大于 `8.0`
- 评分人数大于 `3000`
- 条目是首次达标，或者在冷却后出现了明显增长，值得再次提醒

这个 skill 适合周期性运行。它会把最新快照和本地状态文件做比对，从而识别出：

- 新增命中的高分条目
- 已命中过、但后续涨幅明显的条目
- 仍在观察中、还未达标的条目

## 适用场景
当用户想做这些事情时使用这个 skill：

- 监控豆瓣近期新冒出来的高分内容
- 避免同一个条目反复提醒
- 在热度继续上升时发出二次提醒
- 自动生成一份简洁的周期性监控报告

## 输入参数
运行前收集或默认使用这些参数：

- `min_rating`
  默认 `8.0`
- `min_rating_count`
  默认 `3000`
- `categories`
  默认 `["movie", "tv", "variety", "documentary", "animation"]`
- `state_path`
  默认 `./data/douban-monitor-state.json`
- `only_first_alert`
  默认 `true`
- `realert_cooldown_days`
  默认 `7`
- `rating_delta_for_realert`
  默认 `0.3`
- `rating_count_delta_for_realert`
  默认 `5000`
- `milestone_counts`
  默认 `[10000, 30000, 100000]`

如果用户没有指定候选源，就默认合并少量可靠入口页和热榜接口，组成一个候选池。

## 数据源策略
采用“豆瓣侧 + TMDB 侧”的混合候选池。

默认候选源：

- 豆瓣移动端周榜页，例如 `https://m.douban.com/subject_collection/movie_weekly_best`
- TMDB 的电影和剧集热门接口，例如 `trending`、`popular`

这些来源只负责“发现候选”，最终是否命中必须仍然以豆瓣侧数据为准。

推荐角色分工：

- 豆瓣周榜页
  用来发现近期在豆瓣社区热起来的内容
- TMDB 热门接口
  用来补充全球范围内刚冒头、之后可能在豆瓣形成评分规模的内容
- 本地状态文件回查队列
  用来继续关注接近达标但暂未达标的条目

采集后先统一字段结构，再进入后续过滤和比对流程。

如果候选页里没有评分人数等关键字段，先把条目保留在候选池，再通过详情页或 PtGen 补全。

## 输出结构
每次运行都应该输出一份报告，至少包含这些部分：

1. 运行元信息
2. 新增命中
3. 值得二次关注
4. 继续观察
5. 错误或跳过的来源

## 状态模型
为每个归一化后的豆瓣条目保存一条状态记录。优先用 `douban_id` 作为主键；如果拿不到稳定 id，就退回到“标题 + 年份 + 类型”的确定性键。

## 核心规则

### 首次达标
条目满足以下条件时，视为“新增命中”：

- 当前评分大于 `min_rating`
- 当前评分人数大于 `min_rating_count`
- 并且之前没有状态记录，或之前从未达标

### 冷却机制
避免同一条目反复提醒。

- 如果 `only_first_alert` 为 `true`
  默认只提醒一次，除非用户显式开启二次提醒
- 否则
  至少要求距离上次提醒超过 `realert_cooldown_days`

### 值得二次关注
只有在条目已经命中过、且冷却窗口已过时，才判断是否值得再次提醒。满足以下任一条件即可：

- 当前评分比上次提醒时至少高出 `rating_delta_for_realert`
- 当前评分人数比上次提醒时至少多出 `rating_count_delta_for_realert`
- 当前评分人数跨过了 `milestone_counts` 中配置的里程碑

## 执行流程

1. 读取上一次状态文件；如果不存在，就用空状态开始。
2. 构建候选池，来源包括豆瓣周榜页、TMDB 热门接口，以及本地状态里接近达标的条目。
3. 统一字段并尽可能映射到 `douban_id`。
4. 按 `douban_id` 或回退键去重。
5. 把候选分成“当前已达标”和“继续观察”。
6. 与历史状态比对，判定新增命中和二次关注。
7. 更新状态文件和监控库。
8. 输出报告。

## 入库规则
不要把所有发现到的条目都永久加入监控库。只有具备一定潜力的条目，才值得后续继续检查。

入库前至少要满足这些身份条件：

- 有标题
- 有来源链接或稳定外部 id
- 类型已知，或能够归一化到内部类型集

并且至少满足以下任一条件：

- 出现在豆瓣周榜页
- 出现在选定的 TMDB 热门源，并且可以映射到豆瓣身份
- 豆瓣评分大于等于 `7.5`
- 豆瓣评分人数大于等于 `1000`
- 之前已经在监控库里，且仍然处于观察窗口内

## 观察层级

### 高潜
- 豆瓣评分已经大于等于 `8.0`，但评分人数还没到 `3000`
- 或条目已经达标，并且还处于提醒后的观察窗口内
- 默认观察期 `30 天`

### 中潜
- 豆瓣评分在 `7.8 - 8.0`
- 或评分人数已经大于等于 `2000`
- 或仍然持续出现在当前热门源里
- 默认观察期 `14 天`

### 低潜
- 豆瓣评分低于 `7.8`
- 评分人数增长较弱
- 且不再稳定出现在热门源中
- 默认观察期 `7 天`

## 退库规则
出现以下任一情况时，把条目归档：

- 超过观察窗口仍然从未达标
- 豆瓣评分长期低于 `7.5`
- 最近 `7 天` 的评分人数增长低于 `100`
- 连续 `14 天` 没再出现在豆瓣周榜和 TMDB 热门源里
- 身份字段或豆瓣关键字段连续多次解析失败

## 来源优先级
当多个来源字段不一致时，优先级如下：

1. 豆瓣直接页面
2. PtGen 导出的豆瓣侧数据
3. TMDB 元数据

TMDB 负责扩充候选和补展示字段，但不应该覆盖豆瓣评分和豆瓣评分人数。

## 推荐默认值

- 检查间隔：每 `6` 到 `12` 小时一次
- 二次提醒冷却期：`7 天`
- 二次提醒评分涨幅阈值：`0.3`
- 二次提醒评分人数涨幅阈值：`5000`
